# Contributing

Thanks for wanting to improve this skill. A few guidelines to keep it useful.

## What we want

- **Better decision rules** — clearer when to use scheme X vs scheme Y
- **Real examples** — IA patterns applied to real site types (healthcare, e-commerce, SaaS, intranet)
- **Anti-pattern additions** — specific labelling or nav mistakes worth documenting
- **Tool notes** — practical tips from using Optimal Workshop, UXtweak, Maze etc.
- **Bug fixes** — anything that's factually wrong or misleading

## What we don't want

- Pure theory without practical application
- Brand new frameworks that haven't been tested in real projects
- Long additions to SKILL.md — it loads on every conversation and needs to stay tight

## How to contribute

1. Fork the repo
2. Edit the relevant reference file in `skill/references/`
3. Keep the tone: practitioner-first, decision-rule focused, concrete examples
4. Open a PR with a one-line description of what you changed and why

## Skill structure rules

`SKILL.md` is the core file — loaded on every relevant conversation. Keep it scannable. Use tables over prose where possible. Every addition should answer a real decision a designer faces.

Reference files in `skill/references/` are loaded on demand. These can go deeper but should still resolve to actionable guidance, not just descriptions.

## Style

- Write for a working designer, not an academic
- Decisions over definitions
- "Use X when Y" over "X is defined as..."
- Short sentences
- Concrete examples preferred over abstract principles
